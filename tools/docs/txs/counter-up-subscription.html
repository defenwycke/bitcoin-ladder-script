Counter-Up Subscription

Recurring subscription covenant: a counter increments on each payment (max 24). The UTXO is amount-locked to a fixed payment size and auto-renews via RECURSE_MODIFIED. Three spend paths: PAY (increment counter), CANCEL (2-of-2 multisig), or EXPIRE (counter reaches target).

CTR_UP · AMT_LOCK · REC_MOD · SIG
Rung Structure

Three rungs provide three spend paths. The PAY rung enforces the subscription logic: a signature, a counter check, an amount lock, and a recursive mutation that increments the counter on each spend.

Counter Progression

Each PAY spend increments the counter by 1. The COUNTER_UP block is SATISFIED while current < target. When the counter reaches the target (24), the PAY rung locks and the EXPIRE rung becomes active.

How Counter-Up Subscriptions Work
1.
Funding transaction creates 3 rung outputs. The PAY rung contains SIG + COUNTER_UP + AMOUNT_LOCK + RECURSE_MODIFIED. The counter starts at current=0, target=24.
2.
To make a payment, the PAY rung is spent. The evaluator verifies: (a) valid signature, (b) current < target (counter not exhausted), (c) output amount within [min, max] range, and (d) carry-forward conditions match with current incremented by delta.
3.
RECURSE_MODIFIED enforces that the output conditions are identical to the input, except condition field at param_idx=1 in block_idx=1 (the counter's current value) is incremented by delta=1. This is verified at consensus.
4.
AMOUNT_LOCK constrains the output to exactly 25,000 sats (min=max=25000), ensuring each payment is a fixed amount. The fee comes from the 1,000 sat surplus funded above the lock.
5.
After 24 payments, current = target = 24. The COUNTER_UP block becomes UNSATISFIED, locking the PAY rung. The EXPIRE rung (which requires current = 24) becomes spendable. The CANCEL rung requires both parties' signatures and is always available.
Use Cases
ApplicationDescription
Streaming Payments Fixed-amount recurring payments with a hard cap. Each spend releases exactly one payment's worth and re-encumbers the remainder with an incremented counter.
Service Subscriptions Pre-funded subscription where each period's payment requires only a signature. The counter tracks how many periods have been paid. Automatic expiry after the term.
Rate-Limited Withdrawals Treasury UTXO that allows one fixed-size withdrawal per period. The counter caps the total number of withdrawals.
Milestone Payments Contract that releases payments upon milestone completion (each counter increment). Both parties can cancel early via the 2-of-2 CANCEL rung.
